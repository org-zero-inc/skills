# 维度 4：内聚性

核心问题：**职责是否单一、边界是否清晰？**

## 4a. 核心职责识别

用一句话回答：**"这个 Skill 到底在解决什么问题？"**

- 这句话是否简洁明确？（15 字以内能说清？）
- description 的第一句和这句话是否一致？
- 如果描述不出核心职责，很可能内聚性不足。

## 4b. 职责范围评估

- 列举 SKILL.md 中涉及的所有不同任务。
- 这些任务是否属于同一个领域/问题空间？
- 反例：一个 Skill 既做"数据备份"又做"发送周报"——两者不相关，应拆分。
- 正例：一个 Skill 做"PDF 生成、拆分、合并、加水印"——都是 PDF 处理，可以在一起。

**关键判断标准**：如果两个任务可以被不同用户独立使用，就应该拆成两个 Skill。

## 4c. 与其他 Skill 的关系

思考（需要你对当前系统有了解）：
- 是否有其他 Skill 和这个 Skill 做类似的事？
- 如果有，职责边界是否清晰定义？
- 用户在不同 Skill 之间选择时，是否有明确的区分指引？

## 评级标准

| 评级 | 条件 |
|------|------|
| 🟢 | 核心职责明确 + 所有任务属于同一领域 + 与其他 Skill 无重叠 |
| 🟡 | 核心职责可识别但任务范围偏宽 / 与其他 Skill 有模糊重叠 |
| 🔴 | 无法识别核心职责 / 包含多个不相关领域 / 明显职责重复 |
