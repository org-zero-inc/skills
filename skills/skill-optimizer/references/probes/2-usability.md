# 维度 2：可用性

核心问题：**LLM 调用后能否顺利完成任务？**

## 2a. 输入输出明确性

- SKILL.md 是否说明了输入格式（文件类型、参数、数据格式）？
- 是否说明了输出格式（生成什么、保存到哪里、返回什么）？
- 如果有示例，示例是否清晰可理解？

如果 SKILL.md 有脚本，检查脚本的参数定义（argparse 或函数签名）：
- 参数是否有类型说明？
- 默认值是否合理？
- 必填参数和可选参数是否区分清楚？

## 2b. 参数校验与容错

- 脚本是否有对输入参数的校验（类型检查、边界检查、存在性检查）？
- 校验失败时，返回的错误信息是否对 LLM 友好？（例如："日期格式应为 YYYY-MM-DD，请修正后重试" 优于 "Error: invalid date"）
- 对于网络/文件 IO，是否有超时处理？

## 2c. 工作流可执行性

- SKILL.md 中的步骤是否按顺序排列、每个步骤有明确的操作指令？
- 分支情况（成功/失败/超时）是否有说明？
- 对于需要人工判断的节点，是否有明确提示？

## 评级标准

| 评级 | 条件 |
|------|------|
| 🟢 | 输入输出明确 + 参数校验完善 + 工作流步骤清晰 |
| 🟡 | 缺少示例 / 部分校验缺失 / 步骤顺序不够清晰 |
| 🔴 | 输入输出描述缺失 / 无参数校验 / 工作流不可执行 |
