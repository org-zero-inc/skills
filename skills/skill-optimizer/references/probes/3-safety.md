# 维度 3：安全性

核心问题：**是否有不可逆伤害的风险？**

## 3a. 敏感操作识别

扫描 SKILL.md 和 `scripts/` 下的脚本，查找以下敏感操作关键词：
- **数据破坏**：删除、清空、覆盖、DROP、格式化
- **权限变更**：修改密码、授权、封号、踢出
- **资金操作**：转账、扣费、退款、发放
- **对外发送**：发送邮件、推送通知、发布内容

如果发现敏感操作，继续 3b。如果完全没有，跳过 3b，直接评级。

## 3b. HITL 人工确认

对于涉及敏感操作的 Skill，检查是否有以下机制：
- **执行前预览**：是否先向用户展示将要执行的操作？
- **确认步骤**：是否有明确的"确认后才执行"的步骤？
- **可撤销**：操作是否可以回滚？
- **执行后通知**：操作完成后是否通知用户？

在 SKILL.md 中搜索关键词：人工确认、确认后、审批、审核、HITL、Human-in-the-loop、预览、确认才。

## 3c. 输入校验（安全视角）

- 脚本是否对用户输入做了消毒/过滤？（例如：防止命令注入、路径穿越）
- 如果涉及文件操作，是否限制了操作范围（白名单目录）？
- 如果有网络请求，是否有 URL 白名单或校验？

## 评级标准

| 评级 | 条件 |
|------|------|
| 🟢 | 无敏感操作 / 有敏感操作且 HITL 机制完善 + 输入校验到位 |
| 🟡 | 有敏感操作且 HITL 存在但不够完善 |
| 🔴 | 有敏感操作但完全没有 HITL / 存在明显安全检查漏洞 |
